## 2025-03-09 - Fix SSRF Vulnerability in GCS Downloads
**Vulnerability:** The `download_file_from_gcs` function in `qwen3_embed/common/model_management.py` validated GCS URLs using a simple string prefix check (`url.startswith("https://storage.googleapis.com/")`). This allowed an attacker to bypass the restriction via authority manipulation (e.g., `https://storage.googleapis.com@attacker.com/`), causing the `requests` library to interpret the prefix as a username and fetch data from the attacker-controlled server.
**Learning:** Relying on simple string matching (like `.startswith()`) for URL validation is a common anti-pattern that fails to account for how URL parsing libraries handle complex structures, particularly the `user:password@` authority segment.
**Prevention:** Always use a robust URL parsing library (like `urllib.parse.urlparse`) to extract and validate specific URL components (such as `scheme` and `netloc`) explicitly, rather than trying to validate the entire string linearly.
